# LeadCast 画面一覧定義書

`docs/leadcast_requirements.md` の機能要件 F-1〜F-5 を画面単位に分解し、UI 実装の単位として整理したもの。コード上のルーティング設計、Storybook 構成、デザインカンプ作成、テスト計画の起点として用いる。

---

## 0. 画面定義の方針

### 0.1 分類の3軸

| 軸 | 区分 |
|---|---|
| 利用者 | **配信者**（企業ユーザー：管理者／編集者／閲覧者）、**閲覧者**（リード／視聴者）、**公開**（未認証共通） |
| 認証 | 未認証 / 認証済 / 招待リンク経由 |
| フェーズ | **Feature-Phase 1 (MVP) / Feature-Phase 1.5 / Feature-Phase 2 / Feature-Phase 3**（用語定義は `CLAUDE.md` 「フェーズ用語」セクション参照。GTM-Phase とは独立に管理する） |

### 0.2 採番ルール

- 画面 ID: `S-{大分類}-{中分類}-{連番}` 形式
  - 例：`S-PUB-01-01`（公開・LP・トップ）、`S-ADM-04-03`（管理・リード・詳細）、`S-VWR-02-01`（閲覧者・インタラクション・メールゲート）
- 各画面に対応する機能要件番号（F-X-Y）を併記
- 既存の `docs/leadcast_requirements.md` の F 番号を変更しないよう、画面側に紐付け情報を持たせる方針
- 要件書に対応する F 番号がまだ存在しない画面については、関連 F 列に **`F-要追加`** と明記し、§0.5 のオープン課題に列挙する

### 0.3 画面共通要件（全画面で守る）

- **状態定義**：すべての画面で `loading / empty / error / success / disabled / unauthorized` の各状態 UI を持つ
- **日本語 UX**：CLAUDE.md「日本市場固有の UX 配慮」セクションに準拠（敬体エラーメッセージ、姓名分離、住所自動補完、和暦オプション等）
- **アクセシビリティ**：WCAG 2.1 AA / JIS X 8341-3:2016 準拠、キーボード操作完備、スクリーンリーダー対応
- **レスポンシブ**：管理画面はデスクトップ優先（最小 1280px）、閲覧者向けは完全モバイル対応
- **i18n**：すべての文言は next-intl 経由、日本語をプライマリとする

### 0.4 MVP デモサイトと本ドキュメントの関係

リポジトリ直下 `site/` 配下の静的 HTML（GitHub Pages 公開: https://leadcast-lab.github.io/mvp-demo-site1/ ）は、現時点で確定した **MVP のプロダクト・ポジショニング**を視覚化したクリッカブル UI モックである。本ドキュメントの画面 ID 体系はこの MVP デモを正と見なし、整合させる。

MVP デモの中核は「**3 PILLARS**」の三層構成である:

| ピラー | 内容 | 対応する画面 ID 体系（本ドキュメント） |
|---|---|---|
| **1:n 配信** | 1 本の動画を YouTube / X / LinkedIn / Instagram / Facebook + 自社プレイヤーへ一括配信し、プラットフォーム別に視聴・反応・流入リードを横並びで分析 | `ADM-DST` / `VWR-DST`（§5.3, §15.5） |
| **1:1 パーソナライズ** | 反応のあった視聴者へ AI が冒頭挨拶・課題セクションを差し替えた個別動画を生成・配信 | `ADM-PER` / `VWR-PER`（§5.4, §15.6） |
| **Live** | 1:n 公開 Live と 1:few クローズド Live の 2 モードで、視聴中の挙手・チャット・予約ボタンを商談に橋渡し | `ADM-LIV` / `VWR-LIV`（§7.5, §15.7） |

これらは MVP デモで主役級の機能でありながら、`docs/leadcast_requirements.md`（F-1〜F-5）には機能カテゴリとして明文化されていない（一部は F-1-2 動画作成・管理および F-2-3 双方向対話に隣接概念が含まれるのみ）。**要件書への正式な F 番号採番が必要なオープン課題**として §0.5 に列挙する。

### 0.5 要件書（requirements.md）との同期状況

本ドキュメントを更新した時点で、要件書側に存在しない／不整合がある項目を以下に列挙する。要件書を更新する際は本一覧と突き合わせること。

| 項目 | 状況 | 対応案 |
|---|---|---|
| 1:n SNS 一括配信機能 | 要件書に F 番号なし | 新規 **F-1-4「マルチ SNS 配信」** を要件書に追加し、SNS アカウント接続・配信オーケストレーション・プラットフォーム別アナリティクスを定義 |
| 1:1 AI パーソナライズ動画ワークフロー | 要件書では F-1-2 の AI アバターに包含され、ワークフロー観点（自動抽出→生成→配信→追跡）が独立していない | 新規 **F-1-5「1:1 AI パーソナライズ配信」** を追加し、F-1-2（AI 生成）と F-3（リード管理）の橋渡し機能として定義 |
| Live（1:n 公開 / 1:few クローズド） | 要件書では F-1-2 内 Webinar として 1 行のみ。MVP のピラー扱いに対し記述粒度が不足 | F-1-2 を分割し、Webinar と Live を明示分離。あるいは新規 **F-1-6「Live 配信」** として独立 |
| kintone 連携の Feature-Phase | 要件書では Feature-Phase 1.5 で、CLAUDE.md と整合。本ドキュメントの S-ADM-EXT-05 を Feature-Phase 1 から 1.5 へ修正済み | — |

## 1. サイトマップ（俯瞰）

```
LeadCast
├─ 公開（未認証）
│  ├─ マーケティングサイト（LP / 価格 / 事例 / 法令）
│  ├─ サインアップ・ログインフロー
│  └─ 視聴者向け公開ページ（動画 / チャンネル / Live / Webinar）
│
├─ 配信者：管理コンソール（認証必須・テナント別）
│  ├─ ダッシュボード
│  ├─ コンテンツ：動画 / チャンネル / インタラクション
│  ├─ 配信（3 PILLARS）：1:n SNS 配信 / 1:1 AI パーソナライズ / Live
│  ├─ リード管理：リード DB / スコアリング / セグメント / アラート
│  ├─ オートメーション：ワークフロー / トリガー
│  ├─ 連携：CRM / Webhook / API / SNS アカウント
│  ├─ 分析：ヒートマップ / A/B / ROI ファネル / SNS 別 ROI
│  └─ 設定：組織 / ユーザー / ブランド / 課金 / 監査
│
├─ 閲覧者向け（埋め込み or ホスト型）
│  ├─ 動画プレイヤー
│  ├─ インタラクションオーバーレイ群
│  ├─ 1:1 AI パーソナライズ動画受信ページ
│  ├─ Live 視聴
│  └─ チャンネル / Webinar 視聴
│
└─ 共通：エラー / 通知 / メールテンプレート
```
